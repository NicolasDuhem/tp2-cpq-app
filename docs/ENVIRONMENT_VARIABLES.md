# Environment Variables

## Summary

This file maps all environment variables used by TP2-CPQ-App based on static code search of `app/`, `components/`, `lib/`, and `scripts/`.

Do not commit real secrets. Use `.env.example` only for safe placeholders. Variables visible in the provided Vercel screenshot but not found in code are explicitly marked as screenshot-only findings.

## Recommended future improvement

Add a server-only environment validation module that validates required variables at startup or API-call boundaries. The code currently has targeted validation in the CPQ runtime, BigCommerce client, external PostgreSQL client, DB client, and bootstrap script, but there is no single canonical validation module for all environment variables.
